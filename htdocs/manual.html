Taxonomy

Although miRBase names and other annotations should never be considered as definitive, they provide a useful taxonomy for initial quantitative analyses. This section describes how miRBase artifacts are used to construct the different taxonomy levels at which mirUtils reports.

miRBase Artifacts

The mirUtils bundle includes all artifacts from miRBase recent releases in its mirbase sub-directory, which has the following miRBase-defined structure:

  mirbase/
    vNN/           (miRBase version directory, e.g. v21)
      hairpin.fa   (hairpin fasta RNA sequences for all organisms)
      miFam.dat    (family membership file for all organisms)
      genomes/     (per-organism miRNA hairpin/mature names & loci)
        org_prefix.gff3  (e.g. hsa.gff3)

Note that only the miRBase artifacts required by mirUtils functions are shown here, although additional artifacts may be leveraged in the future.

organisms and versions

mirUtils functions refer to miRBase metatdata for a specified organism and in a specified miRBase version. The version names a sub-directory of the mirbase directory where related artifacts are to be found (e.g. mirbase/v21). The organism refers to the prefix of the GFF3 file in the version directory's genomes sub-directory (e.g. mirbase/v21/genomes/).
